第一次安装不需要追求完美。你只需要证明三件事：

- Hermes 已经正确安装
- 至少一个模型 provider 已经配置好
- CLI 回路可以正常工作

## 优先使用官方安装方式

Hermes 官方文档推荐在 macOS、Linux 和 WSL2 上使用一行安装脚本：

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

这条安装路径会帮你处理：

- 本地依赖
- Hermes 运行环境
- 全局 `hermes` 命令
- 第一次 provider 配置

如果安装时跳过了 provider 配置，后面可以单独执行：

```bash
hermes setup
```

## 一次只验证一个 provider

不要同时接三个 provider。第一次只要选一个，先把它跑健康。

官方推荐的两条入口都很直接：

- `hermes setup`
- `hermes model`

多数凭证会写到 `~/.hermes/.env`。第一阶段尽量保持这个文件足够干净。

## 先验证基础 CLI 闭环

安装后，按需要重新加载 shell，再启动 Hermes：

```bash
source ~/.zshrc
hermes
```

你当前只看一个最简单的成功条件：CLI 里能正常得到回复。

只要这一步成立，说明 provider、运行时和命令路径至少都已经健康到可以继续。

## 现在还不要加什么

在基础链路没稳定之前，先不要急着加：

- 消息平台
- voice mode
- Skills Hub 里的可选技能
- 多 agent 委派
- 定时自动化

这些都是第二层能力。第一层先做到稳定、可复现。

## 如果安装过程出错

先走最短排查链路：

```bash
hermes setup
hermes model
hermes doctor
```

`hermes doctor` 很适合用来快速发现缺依赖、环境变量状态错乱和常见配置问题。
