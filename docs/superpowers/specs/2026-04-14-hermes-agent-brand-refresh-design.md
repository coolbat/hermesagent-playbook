# Hermes Agent Brand Refresh Design

## Goal

Reposition this site from an Hermes Agent onboarding hub into a Hermes Agent learning and workflow site while preserving the current content-first information architecture.

## Scope

- Rebrand site language, metadata, and references from Hermes Agent to Hermes Agent
- Refresh the visual system to echo the Hermes Agent logo language
- Keep the current route structure and reading-first layout
- Rewrite the Quick Start, Learn, Template Library, and Troubleshoot article collections in English and Chinese

## Visual Direction

This is a medium-intensity redesign, not a full IA rewrite.

- Preserve the current page hierarchy, card-based reading flow, and navigation structure
- Replace the warm editorial glass style with a darker Hermes-inspired system
- Use black/charcoal backgrounds with bright yellow and orange accents
- Apply pixel-offset and stacked-outline cues to the brand, hero headings, and key callouts
- Keep article reading surfaces calmer than the homepage and Skills page

## IA Strategy

- Keep top-level routes: `/`, `/learn`, `/templates`, `/skills`, `/troubleshoot`
- Keep article collections and most slugs stable unless a slug is explicitly brand-locked to Hermes Agent
- Rewrite the homepage as a Hermes Agent launch path rather than an Hermes Agent onboarding page
- Reframe Skills around the official Hermes skills ecosystem and workflows

## Content Direction

### Quick Start

Base the new flow on official Hermes Agent docs:

- what Hermes Agent is
- installation and setup
- provider/model configuration
- first conversation or first useful workflow
- first-line troubleshooting

### Learn

Shift from generic Hermes Agent onboarding language to Hermes Agent capability building:

- skills and toolsets
- provider setup
- workspace rules and memory
- gateway and messaging
- controlled expansion and observability

### Template Library

Keep templates practical and outcome-oriented, but reframe them as Hermes Agent workflow recipes.

### Troubleshoot Library

Anchor recovery guidance around Hermes primitives such as:

- `hermes setup`
- `hermes model`
- `hermes gateway`
- `hermes doctor`
- provider credentials
- tool and permission boundaries

## Constraints

- Respect existing uncommitted user edits when touching content files
- Avoid a full docs-site clone of the official Hermes website
- Keep bilingual parity across EN and ZH
- Keep the site readable; decorative styling must not reduce scanability
