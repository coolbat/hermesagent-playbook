# Homepage Onboarding Design

## Goal

Refine the homepage into an OpenClaw onboarding product homepage: a page that feels like a high-quality AI/SaaS entry point while preserving the editorial clarity expected from a tutorial site.

The homepage should help new users answer four questions quickly:

1. What is this site?
2. Who is it for?
3. What should I do first?
4. Where do I go when I need deeper learning, starter templates, or recovery help?

## Product Positioning

This is not a general blog homepage and not a pure marketing landing page.

It should feel like:

- a product onboarding hub
- a tutorial-first resource center
- a credible OpenClaw entry point for beginners

It should not feel like:

- a content magazine
- a flashy AI marketing page
- a dense documentation index

## Desired Experience

The page should communicate confidence, clarity, and forward motion.

The user should feel:

- “I know what this is.”
- “I know where to start.”
- “I can get to a first useful result quickly.”
- “If I get stuck, there is a clear recovery path.”

## Layout Direction

Keep the current macro order:

1. Hero
2. Quick Start
3. Learn preview
4. Templates preview
5. Troubleshoot preview
6. FAQ

Do not redesign the homepage into a different information architecture. Instead, redesign each section to behave more like a mature product homepage module.

## Section Design

### Hero

The hero should be a product entry point, not just a branding statement.

The left side should clearly establish:

- what OpenClaw Playbook is
- the primary user type
- the first recommended action
- the value of starting here instead of digging through scattered docs

The right side should become a stronger “first-run control panel” rather than a simple supporting card. It should feel like a structured operational briefing with:

- 30-minute outcome framing
- first-run checklist or status framing
- confidence-building signals

The hero should feel calmer and more precise than it does now. Less poster-like, more product-like.

### Quick Start

Quick Start should feel like a guided onboarding path, not a cluster of cards.

The section should emphasize:

- sequence
- movement
- guided next action

Each step card should behave like a doorway into a tutorial step, with stronger cues around:

- step number
- what this step accomplishes
- what happens next

The supporting overview above the cards should reinforce that this is a linked 5-step system rather than miscellaneous reading material.

### Learn / Templates / Troubleshoot previews

These three sections should no longer feel like visually similar “preview grids.”

They should each read like a different product module:

- Learn: deepen understanding and methods
- Templates: get to a useful output fast
- Troubleshoot: recover from failure through symptoms

The layout system can remain structurally consistent, but the visual treatment and content emphasis should differentiate them.

### FAQ

FAQ remains a trust and reassurance layer near the bottom of the page. It should be visually quiet and supportive, not a competing focal point.

## Visual Direction

Use a hybrid style:

- structural discipline from a high-end SaaS/AI homepage
- reading quality from an editorial knowledge product
- stronger visual identity than a typical enterprise or documentation site

This direction should feel like a **Signal Board**:

- bold enough to be memorable
- structured enough to remain trustworthy
- energetic without becoming a flashy AI landing page

This means:

- clearer component hierarchy
- tighter module responsibilities
- stronger visual contrast between modules
- fewer passive preview grids
- more product-grade clarity and precision

Keep the current warm brand world, serif/sans contrast, and soft-card system, but push them into a bolder, more directional composition.

## Interaction Principles

- Homepage sections should function as routing decisions, not long-form destinations.
- Primary CTA should always be obvious.
- Secondary actions should support user confidence without fragmenting focus.
- Every section should answer “why would I click this?”
- Cross-language layouts should remain stable and avoid fragile headline compositions.

## Explicit Anti-References

The homepage must not feel like:

- a corporate company website
- a documentation portal
- a generic AI landing page full of hype and glow without structure

It should feel like a guided, productized onboarding environment instead.

## Constraints

- Preserve existing route structure and content system.
- Preserve locale switching and theme switching.
- Avoid introducing homepage-only UI that does not scale to the rest of the site.
- Prefer reusable styling improvements over one-off visual hacks.

## Success Criteria

The redesign is successful if:

- the homepage feels more like a premium onboarding product homepage
- Quick Start is the clearest primary path
- Learn / Templates / Troubleshoot feel like differentiated destination modules
- the page remains credible as a tutorial site rather than becoming overly sales-driven
- bilingual layouts stay stable in both EN and ZH
